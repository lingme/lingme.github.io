Github Action 提交站点到百度收录

使用 Hexo 建立博客不知不觉也已经来到第六个年头了，Hexo 也发展成为了轻量级静态博客的首选框架之一，相信使用过 Hexo 的同学都会被他那优雅，快速，以及完整的生态所折服，一般大家都会选择把 Hexo 生成的静态站点部署到 Github Pages，这就会引发一个问题，站点以及文章无法被百度搜索引擎收录。

分析

因为 Github Pages 禁止了百度的爬虫，原因之前 Github 官方也站出来说过，是因为百度爬取频率过高，对他们的服务器造成了压力，影响了用户体验，之后就禁止了百度爬虫访问 Github Pages，所以后来我把站点部署到了 Vercel，因为 Vercel 是允许百度爬虫访问的，我现在的博客是这个样子：

我本地添加了文章以后，推送到 Github 仓库触发 Github Action，工作流会安装 NodeJS，Hexo 之类的环境，然后会运行 Hexo 的生成命令自动部署到 Vercel 的静态托管服务，大家有兴趣可以看一下这篇文章：Github Action 自动部署Hexo

问题

虽然解决了爬虫访问的问题，但是新的问题来了，那就是这么一个小小的博客网站，也没有做 SEO，也没有做引流推广，那得等到何年何月才能等到百度的爬虫，所以只能手动提交站点目录给百度，这样百度就可以第一时间收录（不是提交了就一定收录，但是不提交希望渺茫接近零），那要思考一个问题，如何做到更新一篇博客文章就把站点目录提交给百度，让百度第一时间知道（收不收录那就是百度的事情了，我们无力干涉），接下来就是正文，实现一个 Workflow Pipeline，更新文章的第一时间就要让百度知道我们有新文章上线了。

实践
